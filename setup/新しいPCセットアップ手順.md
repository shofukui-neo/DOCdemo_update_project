# 新しいPCセットアップ手順書（IT初心者向け）

他人のPC（社内協力者・新メンバー等）に DOCdemo_update_project の実行環境を
オンラインで構築するための手順書です。**この手順書は「相手にそのまま渡す」ことを想定**
しています。最後の章「依頼者側チェックリスト」だけ自分用に使ってください。

---

## 0. 全体像（30秒で読める）

| ステップ | 誰が | 所要時間 |
|---|---|---|
| 1. 配布物を受け取る | 相手 | 1分 |
| 2. ZIPを解凍する | 相手 | 1分 |
| 3. 「インストール.bat」をダブルクリック | 相手 | 10〜15分（自動） |
| 4. 完了確認スクリプトを実行 | 相手 | 1分 |
| 5. 動作確認（依頼者と画面共有） | 両者 | 5分 |

**自動でインストールされるもの:** Git / Python 3.12 / VSCode / プロジェクト本体 / Pythonパッケージ / Playwrightブラウザ

---

## 1. 必要なもの（相手のPCの前提）

| 項目 | 条件 | 確認方法 |
|---|---|---|
| OS | Windows 10（1809以降）または Windows 11 | スタート → 「winver」と入力 → Enter |
| インターネット | 接続済み（社内VPN可） | ブラウザで google.com が開ける |
| 管理者権限 | 必要（ソフト3つをインストールするため） | 社内PCなら情シスに事前確認 |
| 空きディスク | 約3GB | エクスプローラー → PC → Cドライブの空き容量 |
| GitHubアカウント | 必要（無料） | https://github.com/signup で作成 |

> **重要**: 社内PCで管理者権限がない場合は、ここで止まります。情シスに「Git / Python / VSCode の3つを winget でインストールしたい」と申請してください。

## 2. 配布方法（依頼者が選ぶ／推奨は方法A）

### 方法A: GitHub からZIPダウンロード（推奨・最速）

依頼者が相手に **このURLだけ** をSlack/メールで送ります。

```
https://github.com/shofukui-neo/DOCdemo_update_project
```

相手側の操作:
1. URLを開く
2. 緑の **「<> Code」** ボタンをクリック
3. 一番下の **「Download ZIP」** をクリック
4. ダウンロードした `DOCdemo_update_project-main.zip` を **デスクトップ** に移動

> 相手にリポジトリの閲覧権限がない場合は、GitHubで `Settings → Collaborators` から相手のGitHubユーザー名を招待してください（依頼者の作業）。

### 方法B: ZIPをそのまま送付（GitHubアクセス不可な相手向け）

依頼者の作業:
1. `setup/installer/` フォルダを丸ごとZIPにする（中に3ファイル: `インストール.bat`, `installer.ps1`, `はじめにお読みください.txt`）
2. Slack / Box / メール添付で相手に送る

相手側の操作:
1. ZIPをデスクトップに保存
2. 右クリック → **「すべて展開」** → 完了

> この方法ではプロジェクト本体は `installer.ps1` 内で `git clone` により別途取得されます。GitHubアカウントは必要です。

## 3. セットアップ実行（相手の操作・超詳細）

### 3-1. ZIPを解凍する

1. デスクトップに置いた ZIP ファイルを **右クリック**
2. メニューから **「すべて展開」** をクリック
3. 表示されたウィンドウで **「展開」** をクリック（保存先はそのままでOK）
4. デスクトップに新しいフォルダができる（例: `DOCdemo_update_project-main`）

### 3-2. フォルダの中の「インストール.bat」を探す

1. 解凍されたフォルダを開く
2. `setup` → `installer` の順にフォルダを開く
3. 中に **「インストール.bat」** というファイルがある

> 拡張子（`.bat`）が表示されない設定の場合は、ファイル名が「インストール」と表示されます。アイコンが歯車のような見た目です。

### 3-3. 管理者として実行

1. 「インストール.bat」を **ダブルクリック**
2. 青い画面で **「WindowsによってPCが保護されました」** と出たら:
   - 「詳細情報」をクリック
   - 下に出てくる **「実行」** ボタンをクリック
3. **「このアプリがデバイスに変更を加えることを許可しますか?」** と出たら **「はい」** をクリック

> もしダブルクリックで何も起きない場合は、ファイルを **右クリック → 「管理者として実行」** を試してください。

### 3-4. ウィザード画面の操作

黒い画面が開き、青枠で「DOCdemo_update_project 自動インストーラー」と表示されます。
以下の3か所だけ入力が必要です。**それ以外は Enter を押すだけ** です。

| 聞かれる内容 | 入力するもの |
|---|---|
| `Enter キーで開始します` | **Enter キー** を押す |
| `プロジェクトをどこに保存しますか?` | **Enter キー**（デフォルトのデスクトップに保存） |
| `GitHubのユーザー名を入力` | 自分の GitHub ユーザー名（例: `shofukui-neo`） |
| `GitHubに登録したメールアドレスを入力` | GitHub登録時のメールアドレス |
| `VSCodeでプロジェクトを今すぐ開きますか?` | `y` を入力して Enter |

> 途中で黄色や赤の文字が出ても、最後に緑の枠で **「インストールが完了しました!」** と出れば成功です。

### 3-5. 完了の合図

最後に以下の緑枠が出れば終わりです。

```
╔═══════════════════════════════════════════════════════╗
║          インストールが完了しました!                  ║
╚═══════════════════════════════════════════════════════╝
```

最後に Enter キーを押すと黒い画面が閉じます。

## 4. 完了確認（相手の操作・1分）

VSCode が自動で開いたら、メニューの **「ターミナル」→「新しいターミナル」** を選びます（または `Ctrl + @` のショートカット）。
画面下に出てきた青い枠（ターミナル）に、以下を **コピペして Enter** します。

```powershell
powershell -ExecutionPolicy Bypass -File .\setup\check_env.ps1
```

以下のようにすべて **緑の `[ OK ]`** が並べば成功です。

```
[ OK ] Git                            v2.x.x
[ OK ] Python                         3.12.x
[ OK ] VSCode                         インストール済
[ OK ] .venv                          作成済
[ OK ] requirements.txt 全パッケージ  9/9
[ OK ] Playwright Chromium            ダウンロード済
```

赤い `[MISS]` や黄色の `[WARN]` が出た場合は **画面のスクリーンショット** を撮って依頼者に送ってください。

## 5. トラブル対応表

| 症状 | 原因 | 対処 |
|---|---|---|
| 「インストール.bat」をダブルクリックしても無反応 | セキュリティで実行ブロック | 右クリック → **「管理者として実行」** |
| 「winget が見つかりません」と出る | Windowsが古い | Microsoft Storeで「アプリ インストーラー」をインストール後、再実行 |
| Python インストール後も認識されない | PATHが反映されていない | **PCを再起動** してから再度 `インストール.bat` を実行 |
| `git clone` で `Authentication failed` | リポジトリにアクセス権限がない | 依頼者にGitHubのCollaborator招待を依頼 |
| `playwright install` がネットワークエラー | 社内プロキシ | コマンドプロンプトで `setx HTTPS_PROXY http://プロキシ:ポート` を設定後、PC再起動 |
| 途中で止まって何も進まない | 不明 | ログを送る: `C:\Users\<自分の名前>\AppData\Local\Temp\DOCdemo_install_*.log` |

ログファイルは Windowsキー + R → `%TEMP%` と入力 → Enter で開くフォルダの中にあります。
